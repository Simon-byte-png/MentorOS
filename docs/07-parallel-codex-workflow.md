# Parallel Codex Workflow

MentorOS is designed for five parallel Codex windows. Each window must respect the root `AGENTS.md` ownership boundaries.

## Recommended Order

1. Window A completes the static A. Quiet Study prototype in `apps/web`.
2. User confirms screenshots and visual direction.
3. Window B prepares schema and repository placeholders.
4. Window C hardens Agent cards and selection policy.
5. Window D designs AI orchestration around the approved frontstage experience.
6. Window E adds evals and acceptance rubrics.

## Current Stop Point

Stop after Window A static prototype and request visual confirmation before backend, AI, or memory persistence work.
