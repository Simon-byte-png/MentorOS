# Frontend Brief: A. Quiet Study

## Direction

**A. Quiet Study** is the confirmed first visual direction for MentorOS.

It should feel like an quiet private intellectual study. The user is not operating a generic chatbot. They are entering a conversation with memory, judgment, disagreement, and a calm sense of depth.

## Core Feeling

- Black-white minimalism
- Large whitespace
- Quiet left navigation
- Wisdom-oriented dialogue on the right
- Lightweight bottom input
- Private intellectual space
- Reading feel
- Low density
- High taste

## Visual Rules

1. Use black, white, and gray as the primary palette.
2. Preserve large whitespace.
3. Use restrained typography with a reading feel.
4. Do not use technology-blue or purple gradients.
5. Do not use large areas of color.
6. Do not create a complex dashboard.
7. Do not fill the screen with stacked cards.
8. Do not create a customer-service chat bubble feeling.
9. Do not use strong gamification.
10. Do not make the page feel like a backend admin system.

## Desktop Layout

1. Left sidebar width: `220px` to `260px`.
2. Conversation column max width: `760px` to `840px`.
3. The right side should preserve whitespace for a future lightweight Memory Hint Rail.
4. The input is fixed to the bottom of the conversation area.
5. The top should show only a light status line, such as `Quiet Council · Memory On`.

## Mobile Layout

1. Collapse the left sidebar.
2. Let the conversation area take the full width.
3. Keep the input fixed to the bottom.
4. Preserve whitespace and avoid crowding.

## Brand Symbol

1. Use a minimal `M` lettermark.
2. Do not create a decorative logo.
3. The `M` may sit in a square or ring, but it must stay restrained.
4. The visual impression should be a private studio, not an AI SaaS template.

## Dialogue Style

1. Do not use large standard chat bubbles.
2. User messages may sit lightly to the right, but must not feel like WeChat.
3. MentorOS and Agent messages should appear as reading blocks.
4. Agent names use small gray type.
5. Body text uses generous line height.
6. Chinese text must not be squeezed into a narrow vertical feeling.

## Suggested Type Sizes

- Page hero title: `48px` to `64px`
- Dialogue body: `17px` to `20px`
- Agent names: `13px` to `15px`
- Sidebar: `14px` to `15px`
- Input: `16px`

## Language Texture

Dialogue copy must feel humane. Avoid templates, report language, and repeated “first, second, finally” structure. The voice should include pauses, judgment, follow-up questions, and disagreement.

## Required Mock Dialogue Roles

- Dialogue Director
- Munger-style Mentor
- Feynman-style Mentor
- Naval-style Mentor
- Taleb-style Mentor

All explanatory product copy must say: **基于公开思想抽象出的认知模型，不代表本人观点。**

## Development Gate

Build a static visual prototype first. Wait for user confirmation before formal frontend development, backend work, AI integration, or memory persistence.
