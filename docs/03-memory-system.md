# Memory System

MentorOS memory is designed to support better long-term thinking. It must remain visible, editable, deletable, and optional.

## Memory Types

- `profile`: durable information about the user that improves context.
- `goal`: long-term goals and desired directions.
- `preference`: preferences about language, interaction, tools, or style.
- `project`: active projects, constraints, and current stage.
- `decision_history`: past decisions and the reasoning behind them.
- `blind_spot`: recurring patterns that may distort judgment.
- `writing_style`: preferred tone, structure, and expression.
- `relationship`: people, collaborators, and relationship context the user explicitly wants remembered.

## User Control

Users must be able to view, edit, delete, or disable memory. The Memory page should communicate control clearly, not hide memory behind settings.

## Over-Memory Rule

Do not write every casual remark to long-term memory. Prefer durable information: goals, preferences, active projects, decision patterns, blind spots, writing style, and explicitly important relationships.
