# Product Principles

MentorOS is a private intellectual dialogue space. The product should help a user slow down a complicated question, retrieve relevant memory, invite distinct cognitive lenses, and form a more grounded judgment.

## Core Principles

1. The user sees conversation, not machinery.
2. The system may structure work internally, but the frontstage must feel natural and humane.
3. Memory exists to help the user think better, not to surveil them.
4. Agent voices are cognitive models abstracted from public ideas, not impersonations.
5. The first version should prove the core feeling before adding infrastructure.

## Current Product Slice

The current slice is a static frontend prototype using mock data. It should communicate the intended experience without pretending that backend, AI, auth, or memory persistence already exists.
